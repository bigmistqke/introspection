Section 2 of 3 — trace file format and event taxonomy

Trace file

One .trace.json per test, written to .introspect/ at test end. Keyed by test title.

.introspect/login--should-redirect-on-success.trace.json
{
  "version": "1",
  "test": {
    "title": "login > should redirect on success",
    "file": "tests/login.spec.ts",
    "status": "failed",
    "duration": 3241,
    "error": "expect(url).toBe('/dashboard') → received '/login'"
  },
  "events": [ ...timeline of events... ],
  "snapshots": {
    "on-error": { ...rich snapshot at moment of failure... }
  }
}

Event types

// CDP-sourced
network.request   url, method, headers, postData
network.response   status, headers, body (truncated at 50kb)
network.error   url, errorText
js.error   message, stack (real frames via Debugger domain)
js.console   level, args (serialized values, not strings)
dom.snapshot   full serialized DOM at a point in time
variable.snapshot   scope chain + evaluated expressions at error

// Browser agent-sourced
browser.click   target selector, text, coordinates
browser.input   target selector, value
browser.navigate   from, to
mark   label, data (custom annotation)

// Plugin-sourced
redux.action   type, payload, stateBefore, stateAfter
react.render   component, props, state
plugin.<name>   arbitrary plugin data

Every event shape

{
  "id": "evt_042",
  "type": "network.response",
  "ts": 1234567891023,      // ms since test start
  "source": "cdp",          // "cdp" | "agent" | "plugin"
  "data": {
    "requestId": "evt_041", // links response to request
    "url": "https://api.example.com/auth/login",
    "status": 401,
    "body": { "error": "invalid_credentials" }
  }
}

Snapshot (on-error)

Triggered automatically on any uncaught error or failed assertion. Captures a richer cross-section than individual events.

{
  "ts": 1234567892100,
  "trigger": "js.error",
  "dom": "...",             // full serialized DOM
  "url": "/login",
  "scopes": [              // CDP Runtime: local vars at each stack frame
    { "frame": "handleSubmit (auth.ts:42)", "vars": { "response": {...} } },
    { "frame": "onClick (LoginForm.tsx:18)", "vars": { "email": "x@y.com" } }
  ],
  "globals": {             // window globals the AI might care about
    "localStorage": { "token": null },
    "location": { "pathname": "/login" }
  },
  "plugins": {
    "redux": { "auth": { "status": "error", "user": null } }
  }
}
