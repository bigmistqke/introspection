Design: Plugin System

Section 3 of 3 — how plugins extend the capture pipeline

Two-sided plugins

Like the core system, plugins have a browser side (accesses app internals) and a server side (transforms events, extends snapshots). Either side is optional.

Plugin interface
interface IntrospectionPlugin {
  name: string

  // Runs in the browser page — access app internals
  browser?: {
    setup(agent: BrowserAgent): void     // subscribe to app state
    snapshot(): Record<string, unknown>  // called on every snapshot
  }

  // Runs in Vite server — transform the stream
  server?: {
    transformEvent(event: TraceEvent): TraceEvent | null  // null = drop
    extendSnapshot(snapshot: Snapshot): Record<string, unknown>
  }
}

Redux plugin example

Browser side hooks into the store via middleware — no changes to app code beyond adding the middleware.

@introspection/plugin-redux
// store.ts — one line
import { reduxMiddleware } from '@introspection/plugin-redux'
const store = configureStore({
  middleware: (m) => m().concat(reduxMiddleware)
})

// Plugin browser side (auto-registered via middleware)
browser: {
  setup(agent) {
    reduxMiddleware.onAction((action, before, after) => {
      agent.emit({ type: 'redux.action',
        data: { type: action.type, payload: action.payload,
                stateBefore: before, stateAfter: after } })
    })
  },
  snapshot() {
    return { state: store.getState() }  // full store in every error snapshot
  }
}

React plugin example

Uses __REACT_DEVTOOLS_GLOBAL_HOOK__ — no app code changes needed at all.

@introspection/plugin-react
browser: {
  setup(agent) {
    // hooks into React's internal fiber via DevTools hook
    // only emits on error boundaries and uncaught render errors
    // (not every render — too noisy)
    hookReactErrors((error, componentStack) => {
      agent.emit({ type: 'react.error',
        data: { error, componentStack } })
    })
  },
  snapshot() {
    return { tree: serializeComponentTree() }  // in error snapshots only
  }
}

Registration

// vite.config.ts — server side registered here
import { introspection } from '@introspection/vite'
import { reduxPlugin } from '@introspection/plugin-redux'
import { reactPlugin } from '@introspection/plugin-react'

export default defineConfig({
  plugins: [
    introspection({
      plugins: [reduxPlugin(), reactPlugin()],
      capture: {
        ignore: ['react.render'],
        network: { ignoreUrls: [/\.(png|jpg|woff)/] }
      }
    })
  ]
})

Custom plugin

Any app-specific state can be captured. Example: capturing a custom router's state.

const routerPlugin: IntrospectionPlugin = {
  name: 'router',
  browser: {
    setup(agent) {
      router.on('navigate', (from, to) => {
        agent.emit({ type: 'plugin.router',
          data: { from, to, params: to.params } })
      })
    },
    snapshot: () => ({ route: router.current })
  }
}
